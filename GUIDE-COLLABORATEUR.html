01 — Introduction

Aperçu du projet

GTA 6 Hub est un fansite francophone dédié à Grand Theft Auto VI, prévu pour le 19 novembre 2026. Le site agrège automatiquement des actualités anglophones, les traduit en français, et les publie. Il propose également des pages de contenu fixe : personnages, map, config PC, trailers, etc.

Le site est entièrement en production — il est live, indexé par Google, et alimenté automatiquement chaque jour à 8h UTC par un pipeline de collecte d'articles.

💡 Pour qui est ce guide ?

Pour toute personne amenée à travailler sur le projet : ajout de contenu, amélioration SEO, nouvelles fonctionnalités. Aucune connaissance préalable en développement n'est requise pour les sections contenu et SEO éditorial. Les sections techniques sont marquées avec un niveau de difficulté.

DeepL — la traduction automatique

DeepL est l'API de traduction utilisée par le pipeline pour convertir les articles anglais en français. C'est l'une des meilleures IA de traduction disponibles — bien supérieure à Google Translate pour le français.

Comment ça fonctionne

Chaque nuit à 8h UTC, le pipeline récupère les articles des sources anglaises, les envoie à DeepL, et stocke les versions françaises dans Sanity. Les articles francophones (ActuGaming, JeuxVideo.com…) ne sont pas traduits.

Limites du plan gratuit

Le plan Developer gratuit de DeepL permet 500 000 caractères par mois. Avec ~10 articles/jour de ~3 000 caractères, ça représente ~900 000 caractères/mois. Si la limite est atteinte, les articles seront stockés en anglais jusqu'au mois suivant.

⚠️ Surveiller la consommation

Consulter le dashboard DeepL (deepl.com) une fois par mois pour vérifier la consommation. Si on approche des 500k caractères, réduire le nombre de sources anglophones temporairement.

04 — Opérations

Le pipeline d'articles automatique

Le pipeline est le cœur du site. Il s'exécute chaque jour à 8h UTC (10h Paris) et effectue les opérations suivantes :

1

Collecte

Interroge 8 sources RSS (Rockstar Intel, GTA BOOM, Push Square, Kotaku, Reddit r/GTA6, Google News GTA 6, ActuGaming, JeuxVideo.com…)

2

Filtre

Ne garde que les articles liés à GTA 6 (détection automatique par mots-clés). Élimine les doublons déjà présents en base.

3

Nettoyage

Supprime le HTML, les bios de journalistes, les mentions RGPD, les bandeaux cookies — ne garde que le texte de l'article.

4

Traduction

Envoie tous les nouveaux articles à DeepL en une seule requête pour traduction EN→FR.

5

Publication

Insère les articles traduits dans Sanity. Ils apparaissent immédiatement sur le site.

05 — SEO Technique

Comprendre le SEO du projet

Le SEO technique concerne tout ce qui aide Google à comprendre et référencer le site correctement — indépendamment du contenu lui-même. Il s'agit de balises HTML, de données structurées, et de performance.

Le site est construit avec Next.js, qui offre des outils natifs pour le SEO. La configuration SEO se trouve dans les fichiers app/*/page.tsx de chaque page.

📍 Où se trouve le SEO dans le code ?

Chaque page du site a une fonction generateMetadata() ou une constante metadata en haut du fichier qui définit le titre, la description et les balises Open Graph. C'est là qu'interviennent la majorité des modifications SEO.

Balises meta — titre et description Intermédiaire

Les balises meta sont les informations que Google affiche dans ses résultats de recherche : le titre bleu et la description grise sous le lien.

Où les modifier

Dans le fichier de chaque page, en haut du fichier app/[page]/page.tsx. Exemple pour la page personnages :

app/characters/page.tsx
export const metadata = {
  title: 'Personnages GTA 6 — Jason, Lucia et le cast complet | GTA 6 Hub',
  description: 'Découvrez tous les personnages confirmés de GTA VI : ...',
}

Bonnes pratiques

  • Titre : 50-60 caractères max, inclure le mot-clé principal au début
  • Description : 150-160 caractères max, phrase d'accroche avec appel à l'action
  • Inclure "GTA 6" ou "GTA VI" dans chaque titre de page
  • Chaque page doit avoir un titre et une description uniques

Pour les articles (dynamique)

Les articles ont une fonction generateMetadata qui récupère le titre et le résumé depuis Sanity automatiquement. Pas besoin de les modifier manuellement — améliorer le titre/résumé de l'article dans Sanity suffit.

app/news/[slug]/page.tsx — exemple
export async function generateMetadata({ params }) {
  const article = await getArticleBySlug(params.slug)
  return {
    title: `${article.title} | GTA 6 Hub`,
    description: article.summary,
  }
}

const jsonLd = {

  '@type': 'NewsArticle',

  image: article.cover_image,

    '@type': 'Organization',
    name: 'GTA 6 Hub',
    logo: { '@type': 'ImageObject', url: 'https://gta6-hub-tawny.vercel.app/logo.svg' }
  },
}

// Dans le JSX de la page :
<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
/>

Sitemap & robots.txt Intermédiaire

Sitemap

Le sitemap liste toutes les URLs du site pour que Google les découvre facilement. Avec Next.js, il se génère automatiquement via le fichier app/sitemap.ts.

app/sitemap.ts — exemple à créer si absent
import { getArticles } from '@/lib/db'

export default async function sitemap() {
  const articles = await getArticles()
  const staticPages = ['/', '/news', '/characters', '/game', '/map', '/config']

  return [
    ...staticPages.map(url => ({
      url: `https://gta6-hub-tawny.vercel.app${url}`,
      lastModified: new Date(),
      changeFrequency: 'weekly',
      priority: url === '/' ? 1 : 0.8,
    })),
    ...articles.map(a => ({
      url: `https://gta6-hub-tawny.vercel.app/news/${a.slug.current}`,
      lastModified: new Date(a._updatedAt),
      changeFrequency: 'monthly',
      priority: 0.6,
    })),
  ]
}

Le sitemap sera accessible à /sitemap.xml et doit être soumis dans Google Search Console.

robots.txt

app/robots.ts
export default function robots() {
  return {
    rules: { userAgent: '*', allow: '/', disallow: '/studio/' },
    sitemap: 'https://gta6-hub-tawny.vercel.app/sitemap.xml',
  }
}

Créer un compte sur search.google.com/search-console et ajouter le site. C'est l'outil indispensable pour suivre le référencement, détecter les erreurs d'indexation et surveiller les performances dans Google.

Performance & Core Web Vitals Intermédiaire

Google intègre la vitesse du site dans son algorithme de classement. Les Core Web Vitals sont les 3 métriques clés :

MétriqueDescriptionCible
LCPTemps d'affichage de l'élément principal< 2.5s
INPRéactivité aux interactions< 200ms
CLSStabilité visuelle (pas de sauts)< 0.1

Comment mesurer

  • PageSpeed Insights — analyse immédiate de n'importe quelle URL
  • Google Search Console → Core Web Vitals — données réelles des visiteurs

Bonnes pratiques images

  • Toujours utiliser le composant <Image> de Next.js (optimisation automatique)
  • Spécifier width et height pour éviter les CLS
  • Ajouter priority uniquement sur l'image hero de chaque page
  • Format WebP ou AVIF quand possible

const SOURCES = [

✅ Trouver l'URL RSS d'un site

L'URL RSS d'un site WordPress se termine généralement par /feed/. Pour les autres CMS, chercher le logo RSS sur le site ou ajouter ?rss=1 à l'URL. Si le site n'a pas de RSS, il n'est pas compatible avec ce pipeline.
